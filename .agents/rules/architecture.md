# architecture

## Directory layout
```
openagents/
├── skills/openagents/        # Single source of truth for the skill
│   ├── SKILL.md              # Router: frontmatter + subcommand table
│   └── references/           # On-demand instruction files (<50 lines each)
├── .agents/rules/            # Project rules (symlinked to .claude/rules)
│   ├── validate.md
│   ├── distributed-skills.md
│   ├── agentskills.md
│   ├── token-efficiency.md
│   └── conventions.md        # generated by `openagents-rules`
├── .claude-plugin/           # Claude Code Plugin Marketplace manifest
├── claude-plugin/            # Plugin manifest (points to ./skills)
├── scripts/                  # validate.sh, clean.sh, publish.sh
├── .github/workflows/        # validate.yml, publish.yml
└── assets/                   # logo, images
```

## Module responsibilities

- **skills/openagents/**: the only place the skill's instructions live.
- **.agents/rules/**: conventions + guardrails consumed by agents (and the
  skill itself when generating rules).
- **.claude-plugin/ + claude-plugin/**: distribution manifests for the Claude
  Code Marketplace / Plugin system.
- **scripts/**: all executable logic (validation, cleanup, publish gate).
- **.github/workflows/**: CI (validate) and CD (tag + release).

## Boundaries

- `skills/openagents/` is the single source of truth — never duplicate its
  content elsewhere; symlink (`.claude/rules -> ../.agents/rules`) instead.
- Distribution manifests describe the skill; they must not contain
  instructions (those live in `SKILL.md` + `references/`).
- Released artifacts are created by CI only — never push tags/releases by hand.
